ricorsione
La ricorsione nel λ calcolo può essere realizzata definendo una funzione
$$Y = (\lambda y.(\lambda x.y(xx))(\lambda x.y(xx)))$$

Questa funzione applicata ad una generica funzione R produce:
$$YR = (\lambda x.R(xx))(\lambda x.R(xx))$$
Che ridotta diventa:
$$R((\lambda x.R(xx))(\lambda x.R(xx)))$$
Ma questo significa che $YR = R(YR)$ che é la funzione R é valutata ricorsivamente usando Y R come primo argomento.

Possiamo definire:

$$R = (\lambda rn.Z n \, 0(nS(r(Pn))))$$

Questa definizione verifica n, se é zero il risultato della funzione é 0. Se n é diverso da zero allora la funzione successore, o meglio somma se applicata infissa, (S) é applicata n volte all'applicazione di r (ricorsione) con argomento il predecessore (P) a n.

Come si fa ad assegnare ad r nell'espressione sopra la chiamata ricorsiva a R dato che le funzioni nel λ calcolo non hanno nome?

Quello che segue é il programma del quicksort scritto in Haskell (linguaggio funzionale)

```haskell
qsort [] = []
qsort (p:xs) = qsort [x | x <-xs, x <p] ++ [p] ++ qsort [x | x <-xs, x >= p]
```

Difficilmente è possibile scrivere in altro linguaggio QuickSort su una riga. Non vi sono assegnamenti, vettori, indici, variabili, ecc.

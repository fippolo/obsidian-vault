### Gradle
Gradle è un sistema avanzato per la gestione del processo di compilazione, general purpose, basato su Groovy e Kotlin.
Groovy e Kotlin nascano da Scala e vengono eseguiti da la Java VM e possono integrare tutte le librerie standard di Java.
Gradle supporta il download automatico e la configurazione di dipendenze di altre librerie.
Sono supportati i repository Maven ed Ivy da dove è possibile recuperare le librerie necessarie.
Questo consente di riusare gli artefatti di sistemi di building esistenti. Gradle supporta il build di progetti ed artefatti multipli.
per esempio:
- progetto core
- progetto desktop app
- progetto android app
- ecc...
ciò ci rende in grado di fare il deploy su vari dispositivi utilizzando il progetto adeguato
è basato sulla nozione di progetto e task. i task sono le attività specifiche che noi dobbiamo eseguire quando devo fare il building di un applicazione.

quando abbiamo un obbiettivo si inizia sempre con due progetti :
- il progetto libreria
- il progetto app
Un Gradle build è costituito da uno o più project.
Ogni project rappresenta qualcosa che deve essere compilato o una attività che deve essere eseguita.
Ogni project è a sua volta costituito da un insieme di task.
Ogni task rappresenta un pezzo di attività di lavoro che il build deve eseguire, come ad esempio compilare i codici sorgente, eseguire i test o generare i Javadoc.

### setup di un progetto
Creiamo quindi una directory dove il nostro progetto dovrà essere salvato.
Possiamo quindi utilizzare il comando Gradle init per generare un semplice progetto:
```bash
gradle init
```

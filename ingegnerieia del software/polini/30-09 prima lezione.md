# Obbiettivi del corso
### Autonomia di giudizio

- Gli studenti acquisiranno competenze nel valutare:
  - l'efficacia e l'efficienza delle soluzioni software proposte,
  - le tecnologie e le architetture più adeguate per i diversi contesti applicativi associati allo sviluppo software.

### Capacità di comunicare

- Gli studenti saranno in grado di:
  - redigere documentazione tecnica,
  - presentare e discutere le soluzioni software adottate,
  - gestire la comunicazione in contesti di sviluppo software all'interno del team.

### Capacità di apprendimento

- Gli studenti acquisiranno la capacità di:
  - aggiornarsi costantemente, individuando autonomamente nuove tecnologie e approcci emergenti nel campo dell'ingegneria del software.
  - di adattare e ampliare le proprie competenze in funzione delle esigenze del mercato o del progetto.
# Contenuti del corso: Teoria

1. Introduzione e caratterizzazione dell'Ingegneria del Software
2. Processi di Sviluppo Software
    - Introduzione ai processi strutturati per lo sviluppo efficiente.
3. Il Processo di Sviluppo Unificato (UP)
    - Approfondimento del framework UP per la gestione e l'organizzazione del ciclo di vita del software.
4. Ingegneria dei Requisiti
    - Analisi e gestione dei requisiti per garantire che il software soddisfi le esigenze degli stakeholder.
5. Definizione di Scenari e Casi d'Uso
    - Tecniche per descrivere funzionalità e requisiti tramite scenari e casi d'uso.
6. UML e il suo Uso nello Sviluppo di Sistemi Complessi
    - Utilizzo di UML per la modellazione e progettazione di sistemi software complessi.
7. Uso dei Principali Diagrammi UML
    - Diagramma delle Classi
    - Diagramma degli Oggetti
    - Diagramma di Sequenza
    - Diagramma delle Attività
    - Diagramma dei Package
    - Diagramma dei Componenti
    - Diagramma di Deployment
    - Diagramma degli Stati
8. Architetture Software
    - Studio delle strutture fondamentali che compongono un sistema software, incluse le sue componenti e le loro interazioni.

# Cos'è il software?

- Con il termine software in generale si identifica un programma che specifica le istruzioni che un calcolatore dovrà eseguire al fine di raggiungere uno scopo. In questo contesto il termine identifica anche **tutti i documenti** (manuali, file di configurazione, file di istallazione, etc..) che lo descrivono e che sono stati messi a punto durante le **varie fasi della produzione** del sistema. Includiamo nella definizione di software anche le sue caratteristiche strutturali, funzionali, non-funzionali ..., e le sue possibili configurazioni

- l'ingegneria del software è la disciplina che agirà globalmente su tutte le possibili componenti del software al fine di poter produrre software di "qualità"
